1. 제목 		:
					로그인
					
2. 개요			:
					본 유스케이스는 시스템에 진입하기 위해서 로그인을 제공하는 기능이다.
					
3. 관련 액터	:
					관리자, 고객
					
4. 기본 흐름	:
					1) 고객은 시스템에 로그인 화면을 요청한다.
					2) 고객은 아이디와 비밀번호를 입력한다.
					3) 고객은 로그인 버튼을 클릭한다.
					4) 시스템은 고객의 정보를 확인한다.
					5) 회원정보가 맞는 것으로 확인이 되면 메인 화면으로 이동시킨다.
					
5. 대체 흐름	:
					1-A. 고객만 사용할 수 있는 기능을 선택한 경우
						 자동으로 로그인 화면으로 이동시킨다.
					5-A. 회원정보가 없으면 다시 로그인 페이지로 이동 시킨다.
					
6. 예외 흐름	:
					5-A. 데이터베이스 작업이 원활하지 않을 경우 에러페이지로 이동시킨다.

7. 확장점		:
					1) 고객이 아닌 경우 회원가입을 할 수 있도록 버튼을 제공한다.
					2) 회원가입을 하게되면 자동 로그인 되도록 처리한다.
					3) 아이디를 잃어버린 고객을 위해서 아이디 찾기 기능을 위한 버튼을 제공한다.
					4) 비밀번호를 잃어버린 고객을 위해서 비밀번호 초기화 기능을 선택할 수 있는 버튼을 제공한다.
